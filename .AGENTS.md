Offr3D Agent Quickstart
=======================

Context
-------
- Frontend: React 18 + Vite + TailwindCSS.
- Styling: `terminal-*` utility classes (see `src/index.css`) + theme tokens in `tailwind.config.js`.
- Backend: PHP endpoints under `public/api`. Uses MySQL via PDO (`db.php`).
- Settings are managed through `SettingsContext` and persisted via `update-settings.php`.

Local Commands
--------------
- Install deps: `npm install`
- Start dev server: `npm run dev`
- Build production bundle: `npm run build`
- Preview production build: `npm run preview`

Key Paths
---------
- Dashboard: `src/pages/Home.jsx`
- New / edit quote flow: `src/pages/NewQuotePage.jsx`, `src/components/PrintItemList.jsx`, `src/components/QuoteForm.jsx`
- Quote overview / detail: `src/pages/QuotesPage.jsx`, `src/pages/QuoteDetailPage.jsx`
- Settings: `src/pages/SettingsPage.jsx`, `src/context/SettingsContext.jsx`
- Materials: `src/components/settings/MaterialManager.jsx`
- Theme tokens: `tailwind.config.js`
- Custom CSS utilities: `src/index.css`

Working Guidelines
------------------
1. **Styling**  
   - Gebruik bestaande kleuren (`primary`, `accent`, `parchment`, `ink`, …).  
   - Houd `terminal-card`, `terminal-button`, `terminal-pill`, `terminal-note` consistent.  
   - Bewaar neo-retro look: matte beige/charcoal, monospaced fonts, subtiele animaties.

2. **Data Flow**  
   - Quote berekeningen: `calculateItemCost.js` → `calculateQuoteCost.js`.  
   - Settings laden/opslaan via `SettingsContext`.  
   - Offertes CRUD: `save-quote.php`, `update-quote.php`, `delete-quote.php`, `get-quote-detail.php`.

3. **UI Extensions**  
   - Dashboard: houd CTA’s en gegevenskaarten informatief en klikbaar.  
   - Material Manager: statsoverzicht, feedbackmeldingen, modals, filters.  
   - Nieuwe modules zoveel mogelijk baseren op bestaande component en utility patterns.

4. **When in doubt**  
   - Controleer README voor handmatige stappen.  
   - Gebruik `npm run build` voor regressiecheck.  
   - Indien API uitbreidingen nodig zijn, kopieer bestaande endpointstructuur in `public/api`.
